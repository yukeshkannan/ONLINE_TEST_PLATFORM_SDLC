Domain & Email Delivery Integration Plan

A structured plan for professional email communication and domain authentication.

Executive Summary: To establish professional communication with students and clients using official company email addresses (e.g., info@sdlcskills.com) and to render company branding directly inside emails without client-side warnings, we must integrate and authenticate our domain with the Brevo SMTP Delivery Platform. This document outlines the step-by-step roadmap required to configure DNS records and system environments.

1. The Challenge (Why we need domain authentication)

Currently, during local testing, emails are dispatched using sandbox verified accounts. While Brevo accepts custom unauthenticated sender headers, external inbox providers (such as Gmail, Yahoo, and Outlook) immediately flag or drop these emails if the sender domain lacks proper authorization. This verification mechanism is governed by international standard protocols (SPF, DKIM, and DMARC).

By purchasing the domain sdlcskills.com (or any official domain) and establishing cryptographic handshake records in DNS, we guarantee 100% email deliverability straight to the recipient's primary inbox and resolve the issue of broken logo banner images in email templates.

2. Phase-by-Phase Integration Steps

Step 1: Domain Registration & DNS Delegation

We must acquire the domain name (e.g., sdlcskills.com) through an accredited registrar such as Namecheap, GoDaddy, or Hostinger. Once purchased, we gain access to the Domain Name System (DNS) dashboard, where all authorization credentials will reside.

Step 2: Connect the Domain to Brevo

  1. Log into the corporate Brevo Management Console.
  2. Navigate to Senders & IP > Domains > Add a New Domain.
  3. Enter the domain name (e.g., sdlcskills.com) and proceed to generate the verification keys.

Step 3: Add DNS Authentication Records

To authorize Brevo to sign emails cryptographically on behalf of our domain, the administrator must add the following configuration records inside the Domain Registrar's DNS zone editor:

Record Type Host/Name Target Value / Content Purpose
TXT @ (or blank) v=spf1 include:spf.sendinblue.com ~all SPF (Sender Policy Framework): Defines Brevo as an authorized outbound mail server.
TXT mail._domainkey k=rsa; p=MIGfMA0GCSqGSIb3DQEBA... (Provided by Brevo) DKIM (DomainKeys Identified Mail): Encrypts the email headers to prevent tempering.
TXT _dmarc v=DMARC1; p=none; rua=mailto:dmarc@sdlcskills.com DMARC Alignment: Protects the domain from phishing and spoofing.
TXT / HTML @ brevo-code-[unique-id] (Provided by Brevo) Ownership verification: Validates domain ownership.

Step 4: Update Backend Server Configuration

Once DNS replication is complete (usually taking 1 to 24 hours), we will update the server environment configurations (.env file) on the application server host:

  • Change the sender config: SENDER_EMAIL=info@sdlcskills.com
  • Configure client assets URL: CLIENT_URL=https://sdlcskills.com (or production frontend URL)

3. Email Template Branding & Logo Alignment

By hosting the website under the official domain, the logo file is uploaded to the root web assets directory (making it accessible via https://sdlcskills.com/logo.png). This provides the following benefits:

  • Consistent Branding: Email client software (e.g. Gmail) downloads and displays the banner header at the top of verification, registration, and contact emails automatically.
  • Spam Protection: Verified domains prevent email clients from blocking linked assets, ensuring the logo image loads instantly.
  • Trust and Security: System headers show "Sent by SDLC Platform via sdlcskills.com" which reinforces brand trust.

4. Execution & Support Details

Setting up the DNS changes takes approximately 15 minutes of administrative time. Below are the execution timelines for deployment:

  1. Domain Acquisition: Immediate.
  2. DNS Entry Configuration: 15 minutes.
  3. Propagation Window: 1 - 4 hours.
  4. Server Deploy & Code Update: 10 minutes.
